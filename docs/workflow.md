# Project Workflow: Edge-Cloud Collaborative Time-Series Forecasting

## Phase 1: Cloud-Side Model Training & Parameter Extraction

* Split the selected historical dataset into a training set and a testing set.
* Calculate and extract the global or seasonal normalization parameters (e.g., $\mu$ and $\sigma$) for each sensor feature solely from the training set.
* Transform the continuous time-series training data into sliding windows (flattened lagged features) to represent the temporal context.
* Train a lightweight predictive model (e.g., Random Forest or Logistic Regression) using these flattened time-window features and their corresponding target labels.

## Phase 2: Edge-Side Streaming & Decentralized Normalization

* Utilize the testing set to simulate real-time, streaming IoT data generated by multiple edge devices.
* Simulate a Top-Down synchronization mechanism where the "Cloud" broadcasts the pre-calculated normalization parameters to each simulated "Edge Device" via Over-The-Air (OTA) updates.
* Program each edge device to independently collect incoming streaming data into fixed-size time windows.
* Execute local data preprocessing on the edge devices by applying the received parameters to standardize the data within the window, completely bypassing the need to compute these statistical moments locally.
* Transmit the lightweight, preprocessed data windows from the edge devices back to the central cloud.

## Phase 3: Cloud-Side Sensor Fusion & Real-Time Inference

* Receive the asynchronous data windows from various simulated sensors at the cloud level.
* Execute a Sensor Fusion operation by merging the incoming data streams based on their timestamps, utilizing a defined time tolerance to handle slight synchronization offsets.
* Feed the aligned and fused time-window features directly into the pre-trained forecasting model to generate instant point predictions.
* Route the live predictions and the actual ground-truth labels to a real-time interactive dashboard (e.g., built with Streamlit) to dynamically visualize the model's performance and the simulated edge-cloud data flow.
